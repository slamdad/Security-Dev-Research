Python Errors & Debugging 

⸻

1. Common Python Errors

1. SyntaxError

Occurs when Python cannot understand the code structure.

print("Hello, world!"
# SyntaxError: unexpected EOF while parsing

Fix: Close the parenthesis.

print("Hello, world!")

⸻

2. NameError

Occurs when a variable is not defined.

print(name)
# NameError: name 'name' is not defined

Fix:

name = "John"
print(name)

⸻

3. TypeError

Occurs when operations are done on incompatible data types.

5 + "5"
# TypeError: unsupported operand type(s)

Fix:

print(5 + int("5"))

⸻

4. IndexError

Occurs when accessing an invalid index in a list/string.

my_list = [1, 2, 3]
print(my_list[5])
# IndexError: list index out of range

Fix:

print(my_list[2])

⸻

5. AttributeError

Occurs when calling a method that doesn’t exist for that object.

num = 42
num.append(5)
# AttributeError: 'int' object has no attribute 'append'

Fix:

my_list = [42]
my_list.append(5)

⸻

2. Debugging Techniques

A. Using print() and f-strings

Helps track variable values and flow.

def add(a, b):
    result = a + b
    print(f"Adding {a} and {b} gives {result}")
    return result
add(2, 3)

⸻

B. Using pdb (Python Debugger)

import pdb
def divide(a, b):
    pdb.set_trace()
    return a / b
print(divide(10, 2))

Useful pdb Commands

* n → next line
* s → step into
* c → continue
* p variable → print value
* whatis variable → type check

⸻

C. VS Code Debugging

Steps:

1. Add code:

def divide(a, b):
    result = a / b
    return result
print(divide(10, 2))

2. Set breakpoint (click left side)
3. Press F5 to start debugging

Key Controls:

* F5 → Continue
* F10 → Step Over
* F11 → Step Into
* Shift+F11 → Step Out

⸻

3. Exception Handling

Used to prevent crashes and handle errors safely.

try:
    x = 10
    y = 0
    print(x / y)
except ZeroDivisionError:
    print("Cannot divide by zero")

⸻

4. Key Takeaways

* Error messages tell what went wrong and where
* Debugging helps find and fix issues
* Exception handling helps manage runtime errors
* Use:
    * print() → quick checks
    * pdb → deep debugging
    * IDE tools → visual debugging
