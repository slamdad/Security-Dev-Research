# 1. Linux Basics — navigation & filesystem

## File & Directory Commands

- `ls` – List directory contents
- `cd folder` – Change directory
- `cd ..` – Move up one directory
- `pwd` – Print working directory
- `mkdir test` – Create new directory
- `rm file.txt` – Remove file
- `rm -r folder` – Remove directory recursively
- `touch file.txt` – Create empty file (or update mtime)
- `nano file.txt` – Edit file with nano
- `mv a b` – Move or rename
- `cp a b` – Copy file

## `ls` variants

- `ls -l` — long listing with permissions, owner, size, date
- `ls -lh` — human-readable sizes (KB/MB/GB)
- `ls` — short listing

## Notable directories (short)

- `/etc` — system configuration files
    - notable: `sudoers`, `passwd`, `shadow`
- `/var` — variable data (logs, mail, db files)
    - `/var/log` — system and service logs (e.g., `/var/log/syslog`)
    - `/var/tmp` — persistent temporary files
- `/root` — home directory of root user (not `/home`)
- `/tmp` — temp directory writable by all; usually cleaned on reboot

# 4. Disk & process management

- `df -h` – Show disk space in human form
- `du -sh *` – Directory sizes summary
- `top` – Interactive process monitor
- `ps aux` – List all processes (all users)
- `ps` – list current user's processes only
- `kill PID` – Terminate process

## Process signals

- **SIGTERM** (kill -15): graceful stop (cleanup allowed)
    - `kill -15 <PID>`
- **SIGKILL** (kill -9): force kill (no cleanup)
    - `kill -9 <PID>`
- **SIGSTOP**: suspend/pause a process
    - `kill -SIGSTOP <PID>`

# 5. Searching files & content

- `find . -name file.txt` — search by name
- `locate file.txt` — fast search (updatedb index)
- `grep "text" file.txt` — search within files
- `grep -E` — extended regex (e.g., `grep -E "Failed password|Invalid user" /var/log/auth.log`)

Useful `find` examples:

- `find . -readable -size 1033c ! -executable` — readable, 1033 bytes, not executable
- `find / -user bandit7 -group bandit6 -size 33c` — find file owned by bandit7, group bandit6, size 33 bytes

Other utilities:

- `strings data.txt` — extract printable strings from binaries or mixed data
- `sort data.txt | uniq -u` — lines that appear exactly once
- `cut -d ":" -f 1 /etc/passwd` — list usernames from `/etc/passwd` (better than `cat | cut`)

# 7. Networking, SSH & file transfer

## SSH

- Connect: `ssh user@ip` (example `ssh root@192.168.1.10`)
- Use custom port: `ssh -p 2222 user@host`
- Logout: `exit` or `Ctrl + D`

## SCP

- Send file to remote:
    - `scp important.txt ubuntu@192.168.1.30:/home/ubuntu/transferred.txt`
- Get file from remote:
    - `scp ubuntu@192.168.1.30:/home/ubuntu/documents.txt notes.txt`

## wget & starting simple server

- Download file: `wget http://example.com/file.txt`
- Custom port example: `wget http://10.10.8.157:8000/myfile`
- Serve current folder: `python3 -m http.server`
- Download from another machine (when hosting with `python -m`): `wget http://[Your_IP_Address]:8000/filename`

## OpenVPN

- Run VPN: `sudo openvpn config.ovpn`
- Stop VPN: `Ctrl + C`
- Check: `ip a` and look for `tun0`

# 8. Job control, background & operators

- `&` — run command in background (e.g., `script.sh &`)
- `jobs` — show background/suspended jobs
- `Ctrl+Z` — suspend a running process (sends to background stopped)
- `fg` — bring job to foreground
- `&` example: `ping google.com &`

Pipe & operators:

- `|` — pipe output to another command (e.g., `cat file.txt | grep hello`)
- `&&` — run second command only if first succeeds (`mkdir test && cd test`)
- `>` — write/overwrite file (`echo hi > file.txt`)
- `>>` — append to file (`echo again >> file.txt`)

# 10. Logs — Linux (focus)

- `/var/log/syslog` — general system messages (services, kernel messages depending on distro). Good for service crashes, cron job output, systemd, daemon messages.
- `/var/log/auth.log` (or `/var/log/secure` on RHEL) — authentication events: logins, su/sudo, sshd, PAM, failed/successful auths.

Useful quick commands:

```bash
sudo tail -n 200 /var/log/syslog
sudo tail -f /var/log/auth.log    # live follow

```

Grep examples:

```bash
# Search for SSH auth failures
sudo grep -i "invalid user\|failed password" /var/log/auth.log

# Show sudo usage
sudo grep -i "sudo:" /var/log/auth.log

# Filter syslog for crash/errors
sudo grep -Ei "error|fail|panic|segfault" /var/log/syslog | tail -n 200

# Extended regex
sudo grep -E "Failed password|Invalid user|pam_unix" /var/log/auth.log

```

Combine with context:

```bash
sudo grep -n "sshd" /var/log/auth.log | tail -n 50
sudo sed -n '100,200p' /var/log/syslog

```

Threat signals to look for:

- Repeated `Failed password` from same IP → brute force
- `Accepted password` from suspicious geolocation or odd hours
- Sudden `crontab` modifications, new services starting, kernel panic traces
- New user added (`useradd`) or password changes (PAM messages)

# 11. `auditd` — syscall & object auditing (Linux)

**Goal:** capture kernel-level audit records for deep forensic trails.

Components & files:

- `auditd` daemon collects kernel audit records
- Rules stored: `/etc/audit/rules.d/*.rules` or `/etc/audit/audit.rules`
- Tools: `auditctl`, `augenrules`, `ausearch`, `aureport`, `aulast`

Common rule examples:

```
-w /etc/passwd -p wa -k passwd_changes
-a exit,always -F arch=b64 -S execve -k execs
-a exit,always -F arch=b64 -S setuid,setgid,setresuid,setresgid -k id_changes

```

Load & query:

```bash
# Temporarily load a rule
sudo auditctl -w /etc/passwd -p wa -k passwd_changes

# Query events
sudo ausearch -k passwd_changes
sudo aureport -f

```

Syscalls to monitor:

- `execve`, `open`, `openat`, `creat`, `unlink`, `chmod`, `chown`, `ptrace`, `mount` — execution, file writes/deletes/persistence
- `connect`, `accept` — network-related (often better captured with network/eBPF tools)

Detection/forensic value:

- `auditd` gives binary executed, args, UID, timestamps — excellent for tracing lateral movement and persistence.

# 12. Sysmon — process/file/network events (Windows)

**Why Sysmon**: high-fidelity process→network→file linking, command-line capture, DLL loads, injection indicators.

Key Sysmon Event IDs:

- **1** — Process Create (includes command line + parent)
- **2** — File creation time changed
- **3** — Network connection (process-level)
- **6** — Driver loaded
- **7** — Image loaded (DLL loads)
- **8** — CreateRemoteThread (injection)
- **11** — File created (detect dropped payloads)
- **12** — Registry value set
- **13** — Registry value deleted
- **22** — DNS query (process-related)

Mapping notes:

- Sysmon File Create / Registry Value Set → Security events 4656/4657 (but richer in Sysmon)
- Network Connection / DNS Query → Security log has no direct alternative; needs firewall/DNS telemetry to complement

Practical:

- Collect `CommandLine`, `ParentImage`, `ImageLoaded`, hashes
- Use curated Sysmon configs and centralize logs to SIEM

# 13. PSReadline & PowerShell history

- `C:\Users\<USER>\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadline\ConsoleHost_history.txt` contains interactive PowerShell history
- **Risk**: may contain sensitive commands or credentials typed interactively
- Mitigations: clear history, disable persistent history, avoid typing secrets interactively

# 14. Web vulnerabilities — XSS, Command Injection, SQL Injection (differences, examples & mitigations)

## XSS (Cross-Site Scripting)

- **What**: inject JS/HTML that runs in victims' browsers
- **Types**: Reflected, Stored, DOM
- **Example**:
    
    ```php
    <p>Hello <?php echo $_GET['user']; ?>.</p>
    
    ```
    
    `?user=<script>alert(1)</script>` → Reflected XSS; script runs.
    
    Redirect example: `<script>window.location='https://google.com'</script>` via encoded parameter will redirect victim browser.
    
- **Why dangerous**: cookie theft, session hijack, CSRF, redirecting to phishing, delivering payloads
- **Mitigations**: output encode/escape, use CSP, templating auto-escaping, HttpOnly cookies

## Command Injection

- **What**: inject OS commands into server-side command execution
- **Example (unsafe PHP)**:
    
    ```php
    $ip = $_GET['ip'];
    $out = shell_exec("ping -c 3 $ip");
    
    ```
    
    If `ip=8.8.8.8; rm -rf /` — remote command execution.
    
- **Why dangerous**: full RCE, data destruction, privilege escalation
- **Mitigations**: avoid shelling out; use APIs; whitelist input; escape with `escapeshellarg()`; use `proc_open()` safely; least privilege

## SQL Injection

- **What**: inject SQL fragments into queries via unsafe concatenation
- **Example (unsafe PHP)**:
    
    ```php
    $user = $_GET['user'];
    $sql = "SELECT * FROM users WHERE username = '$user'";
    
    ```
    
    If `user=admin' OR '1'='1` → bypass auth.
    
- **Why dangerous**: data exfiltration, modification, auth bypass, possible RCE via DB features
- **Mitigations**: parameterized queries / prepared statements (PDO), least-privilege DB accounts, input validation

### TL;DR differences

- **XSS**: code runs in the **user’s browser** (targets other users)
- **Command injection**: attacker runs commands on the **server OS** (targets server)
- **SQL injection**: attacker manipulates **database queries** (targets DB)

# 15. Detection & hunting tips (practical)

## Linux

- Watch for `bash -c '...'`, `python -c`, `perl -e`, `curl|wget` from odd users or crontab-driven downloads
- Grep syslogs and auth logs for suspicious patterns:
    
    ```bash
    sudo grep -E "curl .*http|wget .*http|nc .* -e|bash -c" /var/log/syslog /var/log/auth.log -n
    
    ```
    
- Use `ausearch`:
    
    ```bash
    sudo ausearch -m EXECVE --input-logs | tail -n 200
    
    ```
    

## Windows

- Monitor Sysmon Event ID 1 (Process Create) for `powershell.exe -enc`, `rundll32`, `wmic`, `regsvr32` with suspicious args
- Monitor file drops (Sysmon 11 / Security 4656/4657) in `%APPDATA%`, `C:\Windows\Temp`, `C:\Users\<user>\Downloads`
- Monitor Sysmon 3 (NetworkConnect) to detect C2 with process context

## Regex/grep snippets for hunts

- Linux downloaders in logs:
    
    ```bash
    sudo grep -E "curl .*http|wget .*http|nc .* -e|bash -c" /var/log/syslog /var/log/auth.log -n
    
    ```
    
- Auditd execve:
    
    ```bash
    sudo ausearch -m EXECVE --input-logs | tail -n 200
    
    ```
    
- Windows conceptual searches:
    - search logs for `powershell` AND (`EncodedCommand` OR `IEX` OR `FromBase64String`)

# 17. Hardening & quick mitigation checklist (one-liners)

- Enable Sysmon with curated config and forward logs to central SIEM/ELK
- Enable detailed Windows auditing for process, object access, account management (tune to reduce noise)
- Use `auditd` rules for critical files and `execve` tracking on Linux
- Enforce least privilege for service accounts; rotate credentials regularly
- Sanitize & validate inputs; use prepared statements and avoid dynamic SQL/commands
- Use CSP and HttpOnly/secure cookies for web apps; escape all output
- Protect and rotate command history files; avoid typing credentials interactively
