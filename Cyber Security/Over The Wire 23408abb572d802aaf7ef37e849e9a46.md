# Over The Wire

---

### 🔓 **OverTheWire Wargames Overview:**

1. **Bandit** *(Beginner)*
    
    🔹 Learn basic Linux commands, file permissions, and SSH.
    
    ➤ Best starting point.
    
2. **Narnia** *(Intermediate)*
    
    🔹 CTF-style binary exploitation, especially **buffer overflows** in C.
    
    ➤ Learn stack control and code execution.
    
3. **Leviathan** *(Beginner to Intermediate)*
    
    🔹 Simple binaries, password cracking, basic **reverse engineering**.
    
    ➤ Good for early learners after Bandit.
    
4. **Krypton** *(Beginner to Intermediate)*
    
    🔹 Focus on **cryptography basics**, ciphers, and logic puzzles.
    
    ➤ Helpful if you're new to crypto.
    
5. **Vortex** *(Advanced)*
    
    🔹 Complex **shell scripting**, memory, race conditions.
    
    ➤ Recommended after mastering Narnia/Leviathan.
    
6. **Maze** *(Advanced)*
    
    🔹 System internals, navigation, possibly **custom kernel-level puzzles**.
    
    ➤ Meant for advanced users.
    
7. **Protostar** *(Intermediate to Advanced)*
    
    🔹 Classic **C vulnerabilities** — buffer overflows, format strings, heap issues.
    
    ➤ Great for learning exploitation.
    
8. **Nebula** *(Intermediate)*
    
    🔹 **Linux-specific exploits**, file descriptors, environment abuse.
    
    ➤ Good for real-world exploitation practice.
    
9. **Twisty** *(Intermediate to Advanced)*
    
    🔹 **Reverse engineering** and **logic puzzles** with a twist.
    
    ➤ For puzzle lovers and programmers.
    
10. **Cobalt** *(Advanced)*
    
    🔹 Realistic **advanced exploitation**, including modern memory protections.
    
    ➤ For seasoned exploit devs.
    
11. **Polymer** *(Intermediate to Advanced)*
    
    🔹 Problems in **algorithms and data structures**, some CTF-style.
    
    ➤ Great if you have a CS/programming background.
    

---

### 🔧 Tip for Getting Started:

- Start with **Bandit**, then try **Leviathan** or **Krypton**.
- After that, go to **Narnia** or **Nebula**, depending on whether you like binary or network challenges.
